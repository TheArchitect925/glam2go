# V1 Scope

## V1 Goal
Ship a strong customer-side booking foundation for Glam2GO with enough artist data structure to support real discovery and booking.

## In Scope
- onboarding and authentication baseline
- customer discovery home
- artist search results
- artist profile with portfolio and package visibility
- package/service detail
- booking flow with date, time, and location selection
- transparent pricing including distance/travel policy display
- booking confirmation
- user bookings list and booking detail
- favorites
- customer profile/settings baseline
- localization-ready architecture
- design-system-first theming foundation

## Conditionally In Scope
- light artist-side profile management only if needed to support demoable seed data flows
- checkout/payment placeholder architecture without pretending full payments are complete

## Out Of Scope For V1
- full artist business dashboard
- complex calendar management
- in-app chat
- promotions and referral programs
- admin tooling in the mobile app
- deep review system authoring and moderation
- broad analytics and growth tooling beyond baseline hooks

## V1 Standard
V1 is not a prototype dump. It must have stable architecture, route discipline, localization readiness, and clear booking UX.
